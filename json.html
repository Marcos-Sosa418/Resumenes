¿Qué es JSON?

JSON (JavaScript Object Notation) es un formato de texto sencillo y ligero para el intercambio de datos. Nació como una notación de objetos en JavaScript, pero actualmente es independiente del lenguaje. Se usa ampliamente en APIs y aplicaciones web por su simplicidad y tamaño compacto.

Ventajas de JSON

Ligero

Ocupa menos espacio que formatos verbosos como XML.

Fácil de leer

Su sintaxis es intuitiva y cercana a la de la mayoría de lenguajes.

Amplio soporte

Muchas bibliotecas y lenguajes ofrecen parseo/serialización nativo.

Ideal para la web

Se integra de forma natural con JavaScript y las APIs REST.

Sintaxis: tipos de datos en JSON

  • Números: enteros o decimales (ej.: 123.456).
  • Cadenas: entre comillas dobles (ej.: "Hola").
  • Booleanos: true o false.
  • null: valor nulo (null).
  • Array: lista ordenada entre corchetes (ej.: ["juan","pedro"]).
  • Objetos: pares "clave": valor entre llaves.

Ejemplo de objeto JSON

{
  "nombre": "Juan",
  "edad": 30,
  "activo": true,
  "hobbies": ["leer", "viajar", "nadar"]
}

Modelos de procesamiento y uso

JSON puede ser procesado con distintos modelos dependiendo del lenguaje:

  • Modelo de objeto: carga todo en memoria como un árbol (útil para manipulación).
  • Modelo de flujo: lectura/escritura por eventos (más eficiente para grandes volúmenes).
  • Conversión: bibliotecas convierten JSON en objetos del lenguaje (ej.: Jackson, Gson).

Parseo en JavaScript (seguro)

let texto = '{"nombre":"Miguel","edad":22}';
let datos = JSON.parse(texto);
console.log(datos.nombre); // "Miguel"

Ejemplo clásico con XMLHttpRequest (histórico)

var http = new XMLHttpRequest();
http.onreadystatechange = function () {
  if (http.readyState === 4 && http.status === 200) {
    var datos = JSON.parse(http.responseText);
    console.log(datos);
  }
};
http.open("GET", "datos.json", true);
http.send();

Nota: hoy se suele usar fetch() o librerías (axios, etc.) en lugar de XMLHttpRequest.

Ejemplos comparativos: JSON y su equivalente XML

Ejemplo JSON (menú)

{
  "menu": {
    "id": "file",
    "value": "File",
    "popup": {
      "menuitems": [
        { "value": "New", "onclick": "CreateNewDoc()" },
        { "value": "Open", "onclick": "OpenDoc()" },
        { "value": "Close", "onclick": "CloseDoc()" }
      ]
    }
  }
}

XML equivalente

<menu id="file" value="File">
  <popup>
    <menuitem value="New" onclick="CreateNewDoc()" />
    <menuitem value="Open" onclick="OpenDoc()" />
    <menuitem value="Close" onclick="CloseDoc()" />
  </popup>
</menu>

Comparación JSON vs XML

Característica JSON XML
Tamaño Más compacto Más verboso
Legibilidad Más fácil Más estructurado
Soporte para esquemas Sí (JSON Schema) Sí (XSD)
Uso típico APIs, web, microservicios Documentos, configuraciones complejas
