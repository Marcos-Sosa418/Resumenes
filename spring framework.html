Spring Framework

El Spring Framework es uno de los frameworks más utilizados en el desarrollo de aplicaciones empresariales con Java. Proporciona un conjunto de herramientas que facilitan la creación de aplicaciones robustas, seguras, escalables y fáciles de mantener.

Características Principales

Inversión de Control (IoC)

IoC permite que las dependencias de una clase sean administradas externamente, en lugar de que la propia clase cree sus objetos. Esto promueve el desacoplamiento y facilita las pruebas y el mantenimiento.

Inyección de Dependencias (DI)

Consiste en inyectar las dependencias necesarias en una clase automáticamente. Spring actúa como contenedor que crea y gestiona los objetos del sistema, reduciendo el acoplamiento entre clases.

Spring MVC

Framework basado en el patrón Modelo-Vista-Controlador. Permite crear aplicaciones web estructuradas y organizadas separando la lógica de negocio, la vista y el controlador.

Spring Data

Simplifica la interacción con bases de datos. Abstrae la capa de acceso a datos y soporta tecnologías como JPA, JDBC, MongoDB, Redis, entre otros.

Spring Security

Proporciona un sistema completo de seguridad para aplicaciones Java: autenticación, autorización y protección contra ataques comunes (XSS, CSRF, etc.).

Spring Boot

Herramienta que permite crear aplicaciones Spring con mínima configuración. Incluye servidor embebido, configuración automática y facilidad de despliegue.

Ejemplo: Spring Boot + Controlador Web

A continuación, un pequeño ejemplo de una aplicación Spring Boot con un controlador que devuelve un mensaje.

Clase Principal

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class DemoApplication {
    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}

Controlador (Controller)

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
public class HelloController {

    @GetMapping("/hola")
    public String hola() {
        return "¡Hola desde Spring Boot!";
    }
}

Archivo application.properties

server.port=8080
spring.application.name=demo-app

¿Cómo funciona?

  • La anotación @SpringBootApplication inicializa todo el proyecto Spring.
  • El controlador @RestController expone un endpoint web.
  • @GetMapping("/hola") crea una ruta para acceder mediante navegador o Postman.
  • Spring Boot levanta un servidor interno (Tomcat) automáticamente.
