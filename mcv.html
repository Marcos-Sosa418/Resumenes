Modelo MVC

El patrón de arquitectura Modelo-Vista-Controlador (MVC) divide una aplicación en tres componentes principales: El Modelo, La Vista y El Controlador. Su propósito es separar los datos, la lógica de negocio y la presentación visual, haciendo la aplicación más organizada, reutilizable y fácil de mantener.

Historia

MVC fue introducido por Trygve Reenskaug en los años 70 en Smalltalk-76 y evolucionado en Smalltalk-80. Fue uno de los patrones pioneros en interfaces gráficas modernas separando entrada, procesamiento y salida. Hoy MVC tiene variantes como HMVC, MVP, MVVM, entre otros.

Descripción del Patrón

  • Modelo: Gestiona los datos y la lógica de negocio. Controla consultas, cambios y validaciones.
  • Controlador: Recibe las acciones del usuario, actualiza el modelo y solicita actualizaciones en la vista.
  • Vista: Presenta la información al usuario en formato visual (interfaz).

Ejemplo MVC en Python — Lista de Tareas

Ejemplo práctico usando una To-Do List para ilustrar cómo funciona el Modelo-Vista-Controlador.

Modelo

class TaskModel:
    def __init__(self):
        self.tasks = []

    def add_task(self, task):
        self.tasks.append(task)

    def remove_task(self, task):
        if task in self.tasks:
            self.tasks.remove(task)

    def get_tasks(self):
        return self.tasks

Vista

class TaskView:
    @staticmethod
    def show_tasks(tasks):
        print("To-Do List:")
        for idx, task in enumerate(tasks, start=1):
            print(f"{idx}. {task}")

    @staticmethod
    def show_message(message):
        print(message)

Controlador

class TaskController:
    def __init__(self, model, view):
        self.model = model
        self.view = view

    def add_task(self, task):
        self.model.add_task(task)
        self.view.show_message(f"Task '{task}' added.")
        self.view.show_tasks(self.model.get_tasks())

    def remove_task(self, task_number):
        tasks = self.model.get_tasks()
        if 0 < task_number <= len(tasks):
            removed_task = tasks[task_number - 1]
            self.model.remove_task(removed_task)
            self.view.show_message(f"Task '{removed_task}' removed.")
        else:
            self.view.show_message("Invalid task number.")
        self.view.show_tasks(tasks)

Uso del MVC

if __name__ == "__main__":
    model = TaskModel()
    view = TaskView()
    controller = TaskController(model, view)

    controller.add_task("Learn Python")
    controller.add_task("Write Unit Tests")
    controller.add_task("Build a To-Do App")

    controller.remove_task(2)

Funcionamiento

  • Modelo: TaskModel gestiona la lista de tareas.
  • Vista: TaskView muestra tareas y mensajes.
  • Controlador: TaskController coordina el flujo entre modelo y vista.
